# QR Card Studio

웹 주소, 연락처 명함(vCard), 텍스트, 이메일, 문자, Wi-Fi, 메신저 링크, 위치, 일정, 암호화폐 주소를 QR 코드로 만들고 PNG 또는 SVG로 저장하는 정적 웹페이지입니다.

**바로 사용하기:** https://ko9ma7.github.io/qr-card-studio/

## 특징

- 회원가입과 서버 없이 브라우저 안에서 QR 생성
- 12가지 정보 형식과 실시간 미리보기
- 색상, 점 모양, 프레임, 로고 설정
- PNG 및 SVG 다운로드, 원본 데이터 복사
- 모바일·태블릿·PC 반응형 화면과 다크 모드
- 키보드 탭 이동, 명시적 폼 라벨, 상태 알림 지원

## 로컬에서 확인하기

파일을 직접 열어도 대부분 동작하지만, 브라우저 보안 정책과 실제 배포 환경을 맞추려면 로컬 서버를 권장합니다.

```bash
python -m http.server 8080
```

브라우저에서 `http://localhost:8080`을 엽니다.

## GitHub Pages에 배포하기

1. GitHub에서 새 공개 저장소를 만듭니다.
2. 이 폴더의 `index.html`, `style.css`, `app.js`, `README.md`를 저장소 최상위에 올립니다.
3. 저장소의 **Settings → Pages**로 이동합니다.
4. **Build and deployment**에서 **Deploy from a branch**를 선택합니다.
5. 브랜치는 `main`, 폴더는 `/(root)`를 선택하고 저장합니다.
6. 배포가 끝나면 Pages 화면에 표시되는 주소로 접속합니다.

## 사용 시 주의

- QR은 저장 전에 실제 휴대폰 카메라로 스캔해 결과를 확인하세요.
- Wi-Fi 비밀번호, 연락처, 일정처럼 민감한 정보가 담긴 QR 이미지는 공개 장소에 올리지 마세요.
- 가운데 로고를 크게 넣거나 QR과 배경의 대비를 낮추면 인식률이 떨어질 수 있습니다.
- 암호화폐 지갑 주소와 금액은 전송 전에 원문과 다시 대조하세요.
- QR 생성 라이브러리를 프로젝트에 함께 포함해 배포 후에는 별도 서버나 API 없이 동작합니다.

## 라이선스

페이지 코드의 재사용 조건은 저장소 공개 전에 원하는 라이선스로 정하세요. `vendor/qrcode.min.js`는 Kazuhiko Arase의 MIT 라이선스 `qrcode-generator` 1.4.4를 사용합니다.
